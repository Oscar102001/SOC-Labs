## Recommendations

| Vulnerability | Fix |
| --- | --- |
| Weak login password (brute forced) | Implement account lockout after 5 failed attempts + strong password policy |
| No rate limiting on /login.php | Add rate limiting / CAPTCHA to login page |
| LFI in /stats.php `page` parameter | Validate and sanitize file path input — use allowlists, never pass user input to file functions |
| XSS in /search.php `q` parameter | Encode all user output — use `htmlspecialchars()` in PHP or equivalent |
| Admin panel accessible post-login | Implement MFA for admin access |
| Server fingerprinting via Nmap | Configure server to not expose version headers |
